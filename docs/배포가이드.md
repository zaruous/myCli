# mycli npm 배포 가이드

## 사전 준비

### 1. npm 계정 확인
```bash
npm whoami
```
로그인되어 있지 않으면:
```bash
npm login
```

### 2. 패키지 이름 중복 확인
```bash
npm search mycli
```
이름이 이미 사용 중이라면 `package.json`의 `name` 필드를 고유한 이름으로 변경합니다.  
예: `@zaruous/mycli` (스코프 패키지) 또는 `kyjcli` 등

---

## package.json 점검

배포 전 아래 필드를 확인합니다.

```json
{
  "name": "mycli",           // npm에 등록될 패키지 이름 (전 세계 유일해야 함)
  "version": "1.0.0",        // 배포마다 버전을 올려야 함 (SemVer)
  "description": "...",      // npm 검색 결과에 표시
  "main": "index.js",
  "type": "module",
  "bin": {
    "mycli": "./index.js"    // 전역 설치 시 실행 명령어
  },
  "files": [                 // 배포에 포함할 파일 목록 (없으면 전체 포함)
    "index.js",
    "lib/",
    "search/",
    ".mycli/"
  ],
  "engines": {
    "node": ">=18.0.0"       // 지원하는 Node.js 최소 버전
  }
}
```

### `files` 필드 추가 권장
`node_modules`, `test`, `.env` 등 불필요한 파일이 배포되지 않도록 `files` 필드를 지정하거나 `.npmignore`를 작성합니다.

## .npmignore 작성 (권장)

프로젝트 루트에 `.npmignore` 파일을 생성합니다:

```
.env
.env.*
test/
coverage/
todo.md
test-report.md
*.log
.idea/
.mycli/
```

> `.mycli/` 스킬 디렉터리는 사용자가 직접 구성하는 파일이므로 배포에서 제외하는 것이 일반적입니다.  
> 스킬 예제를 포함하려면 이 줄을 제거하세요.

## 배포 절차

### 1. 테스트 실행
```bash
npm run test:all
```
모든 테스트가 통과해야 합니다.

### 2. 배포 미리보기 (dry-run)
실제로 배포하지 않고 어떤 파일이 포함되는지 확인합니다:
```bash
npm pack --dry-run
```

### 3. 버전 올리기
```bash
# 패치 버전 (버그 수정): 1.0.0 → 1.0.1
npm version patch

# 마이너 버전 (기능 추가): 1.0.0 → 1.1.0
npm version minor

# 메이저 버전 (호환성 변경): 1.0.0 → 2.0.0
npm version major
```

### 4. 배포
```bash
npm publish
```

스코프 패키지(`@zaruous/mycli`)를 **공개**로 배포하려면:
```bash
npm publish --access public
```

## 배포 후 확인

```bash
# npm 레지스트리에서 확인
npm info mycli

# 전역 설치 테스트
npm install -g mycli
mycli
```

---

## 버전 업데이트 배포 (이후 배포)

```bash
# 1. 코드 수정 후 테스트
npm run test:all

# 2. 버전 올리기
npm version patch   # 또는 minor / major

# 3. 배포
npm publish
```

`npm version` 명령은 `package.json` 버전 수정 + git 태그 생성을 자동으로 처리합니다.

---

## 스코프 패키지로 전환하는 경우

`package.json`을 수정합니다:
```json
{
  "name": "@zaruous/mycli"
}
```

그리고 배포:
```bash
npm publish --access public
```

설치 명령도 변경됩니다:
```bash
npm install -g @zaruous/mycli
```

## 문제 해결

| 오류 | 원인 | 해결 |
|------|------|------|
| `403 Forbidden` | 패키지 이름 중복 또는 권한 없음 | 이름 변경 또는 npm 계정 확인 |
| `You must be logged in` | npm 로그인 안 됨 | `npm login` 실행 |
| `Cannot publish over existing version` | 동일 버전 재배포 시도 | `npm version patch` 후 재시도 |
| `index.js` 권한 오류 (Unix) | shebang 파일 실행 권한 없음 | `chmod +x index.js` 실행 |
